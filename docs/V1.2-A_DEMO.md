# GitGov V1.2-A Demo (Jenkins-first MVP)

**Objetivo de la demo:** mostrar trazabilidad `commit -> pipeline` sin romper el flujo base de GitGov.

---

## Qué se demuestra

1. El flujo base sigue vivo (Desktop commit/push + logs/dashboard)
2. Jenkins puede reportar pipeline events a GitGov
3. GitGov correlaciona commits con pipelines por `commit_sha`
4. Dashboard muestra:
- `Commits Recientes` (sin ruido de `attempt_push`/`successful_push`)
- detalle de archivos del commit
- badge `ci:<status>` cuando hay correlación
- widget `Pipeline Health (7 días)`
5. `POST /policy/check` responde en modo advisory

---

## Paso 3 — Enviar pipeline event a GitGov (simulando Jenkins)

### Sin secreto extra (solo admin API key)
```bash
curl -X POST http://localhost:3000/integrations/jenkins \
  -H "Authorization: Bearer TU_API_KEY_ADMIN" \
  -H "Content-Type: application/json" \
  -d '{
    "pipeline_id": "demo-build-001",
    "job_name": "gitgov/main",
    "status": "success",
    "commit_sha": "REEMPLAZAR_SHA",
    "branch": "main",
    "repo_full_name": "MapfrePE/GitGov",
    "duration_ms": 42000,
    "triggered_by": "jenkins",
    "stages": [
      { "name": "Build", "status": "success", "duration_ms": 12000 },
      { "name": "Test", "status": "success", "duration_ms": 24000 },
      { "name": "Deploy", "status": "success", "duration_ms": 6000 }
    ],
    "artifacts": ["gitgov-server-v1.2.0.tar.gz"],
    "timestamp": 1771900000000
  }'
```

### Con secreto Jenkins habilitado
```bash
curl -X POST http://localhost:3000/integrations/jenkins \
  -H "Authorization: Bearer TU_API_KEY_ADMIN" \
  -H "x-gitgov-jenkins-secret: TU_SECRETO" \
  -H "Content-Type: application/json" \
  -d '{ ... }'
```

Respuesta esperada:
- `accepted: true`

## Troubleshooting rápido

### No aparece badge `ci:*` en commit
- Confirmar que `commit_sha` del pipeline coincide con el commit (full SHA preferido)
- Reiniciar server/app y refrescar dashboard
- Verificar `/integrations/jenkins/correlations`

### `401` en Dashboard
- Verificar `Authorization: Bearer` y API key del Control Plane
- Revisar `docs/GOLDEN_PATH_CHECKLIST.md`

### `duplicate=true` al primer intento Jenkins
- Revisar `pipeline_id`, `job_name`, `commit_sha`, `timestamp`
- El dedupe usa combinación de esos campos

### Jira devuelve `401`
- Verificar `Authorization: Bearer ...`
- Si configuraste `JIRA_WEBHOOK_SECRET`, enviar `x-gitgov-jira-secret`

### `correlations_created = 0`
- Verificar que el commit message o branch contiene un ticket (`PROJ-123`)
- Ejecutar de nuevo `/integrations/jira/correlate` ampliando `hours`
- Verificar que existan commits `event_type=commit` en `/logs`
